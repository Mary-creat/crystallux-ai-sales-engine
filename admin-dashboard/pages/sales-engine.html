Live call
Real-time Vapi call state. Wired in follow-up commit.

What this will show

Active outbound / inbound calls in flight. For each call: lead, advisor, elapsed time, current transcript chunk, and the script suggestion (if any) currently displayed on the advisor's screen.

Data source: clx-agent-voice-outbound-v1 + clx-agent-voice-inbound-v1 + clx-vapi-transcript-streamer-v1 + clx-realtime-script-suggester-v1. The transcript streamer writes to call_event_log; this panel will poll that table every 14 s.

Activation path: Mary configures the Vapi credential in n8n + activates the four workflows above. This panel will detect the live row and start rendering.

Script suggestions
Closing / objection / discovery scripts surfaced in real time during an active call.

What this will show

The newest unresponded suggestion from script_suggestion_log for the active call, ranked by confidence. Advisor accepts / rejects → workflow logs back into the same table for the learning loop (clx-script-learning-loop-v1).

  • Discovery — open-ended questions for first 10 minutes
  • Objection handling — surfaced when buy/closing signals dip
  • Close — surfaced when buy signals spike + 15 min mark

Data source: script_suggestion_log + closing_scripts (defined in db/migrations/missing-table-fixes.sql).

Post-call review
Last 20 completed calls with sentiment + closing / buy signals + script-suggestion acceptance rate.

What this will show

Each completed call: lead name, advisor, duration, closing_signals_detected, buy_signals_detected, script_suggestions_shown / accepted, and the analyzer's outcome tag (set by clx-post-call-analyzer-v1).

Data source: call_event_log + agent_decisions. The post-call analyzer writes an outcome row within ~60 s of hang-up; this table reads back over the last 24 h.
